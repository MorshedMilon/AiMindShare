# M16 Campaigns Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship M16 Campaigns (Session 17) as a vertical slice to Definition-of-Done: a block-based email builder, broadcasts + drip sequences run as `jobs` fan-out, A/B subject testing, global suppression + one-click unsubscribe (M05), open/click tracking, a live SendGrid adapter (ready-not-run), per-send metering, live SMS steps via M12 ŌĆö with the AI copywriter, domain-auth verify, spam-score API, and MJML library as honest flagged scaffolds.

**Architecture:** Locked stack ŌĆö vanilla HTML/CSS/JS front end + Supabase (Postgres + RLS + Edge Functions + Vault + Realtime + pg_cron + `public.jobs`). Sends are a `campaign.send` fan-out job ŌåÆ throttled `email.deliver`/`sms.deliver` batch jobs; drips are `run_after`-delayed `sequence.step` jobs (the D-061 WAIT pattern) plus an hourly reconciliation cron. SendGrid runs REST + Web Crypto (ECDSA event-webhook verify) with the key in Vault (D-028 pattern). Sends/opens/clicks publish to M13's `emit_trigger` bus and M09's `log_activity` timeline and meter `email`/`sms` in the success transaction.

**Tech Stack:** PostgreSQL (migration `0020`), Supabase Edge Functions (Deno/TypeScript), pg_cron + `public.jobs`, Node worker handlers, PGlite verification probe (Node ESM), vanilla HTML/CSS/JS with `tokens.css`/`app.css`/`components.css`.

**Verification model (repo convention, not pytest):** DB/RLS/pipeline logic is verified by `workers/verify/m16probe.mjs` against real Postgres via PGlite; front end by the local preview server; Gate-8 by `scripts/gate8.sh`. "Failing test first" here = write the probe assertion, run it red, implement the SQL/logic, run it green. **This workspace is not a git repo** (`git rev-parse` fails) ŌĆö treat every "Commit" step as optional: run it only if git is initialised, else skip and record progress in TASKS.md.
