# Plagiarism/AI-detection check + Auto-Rewrite Loop — design

**Date:** 2026-07-13
**Status:** approved, not yet implemented

## Problem

Content generated by M22-auto (and any future pipeline) needs a way to flag
likely-plagiarized or likely-AI-written passages, and to automatically try to
fix them before publish. This spec builds the first real `plagiarism`
capability adapter on top of the already-merged provider abstraction layer
(`workers/config/providers.js` — `resolveProvider`, `logProviderUsage`,
`PROVIDER_CONFIG`), plus an orchestration loop that rewrites flagged content
via Claude until it's clean or attempts run out.

## Explicitly out of scope (this session)

- Real BYOK adapters for Copyleaks, Originality.ai, GPTZero, or Winston AI.
  These are registered in `PROVIDER_CONFIG.plagiarism.paid` (name + env var +
  description) so `resolveProvider` can route to them later, but no adapter
  file is implemented. Selecting one today returns an honest
  `unavailable / adapter_not_implemented` result — same posture as D-063.
- Any DB/Vault wiring for the plagiarism check itself. `checkOriginality` is a
  pure function of `(text, config)`. Only the rewrite loop touches Vault, and
  only by reusing the existing `callAnthropicForArticle(db, workspaceId, ...)`
  helper from `workers/llm.mjs` unchanged.
- Any web-search-based or crawled-corpus plagiarism checking. The free tier
  has no external API and no key — see "Comparison corpus" below for what it
  checks instead.

## `checkOriginality(text, config = {})` — `workers/providers/plagiarism.js`

1. `const { tier, provider } = resolveProvider('plagiarism', config)`.
2. If `tier === 'paid'` (a name is registered but has no adapter): return
   `{ plagiarismScore: null, aiLikelihoodScore: null, flaggedSentences: [], provider: provider.name, unavailable: true, reason: 'adapter_not_implemented' }`.
   Never throws — matches `workers/llm.mjs`'s discriminated-result convention.
3. If `tier === 'free'`, run the local computation (below).
4. Every call (free or paid-unavailable) calls
   `logProviderUsage('plagiarism', provider.name, { tier, plagiarismScore, aiLikelihoodScore })`.

### Local computation

- Split `text` into sentences (simple regex on `. ! ?` boundaries, keep
  originals for output). Tokenize/normalize each (lowercase, strip
  punctuation) for scoring only.

- **Comparison corpus.** If `config.corpus` (array of `{id, text}`) is
  supplied, each input sentence is compared against every corpus sentence via
  TF-IDF cosine similarity, and its score is the best match found. If no
  corpus is supplied, each input sentence is instead compared against every
  *other* sentence in the same text — i.e. internal near-duplicate/redundancy
  detection. This means the check is never a no-op even with zero external
  input, but it also means "plagiarism" here really means "text matches a
  caller-supplied reference corpus, or repeats itself" — not "matches
  something somewhere on the web." Document this limitation clearly in
  `README-providers.md`.

- **`plagiarismScore` (0–100).** A sentence is "flagged" once its best-match
  similarity crosses `config.sentenceThreshold` (default `0.5`).
  `plagiarismScore = round(100 * totalWordsInFlaggedSentences / totalWords)`
  — the percentage of the document that's likely copied or duplicated.

- **`aiLikelihoodScore` (0–100).** A heuristic combining:
  - *Burstiness*: coefficient of variation (stdDev / mean) of sentence
    lengths in words. Human writing tends to vary sentence length more; low
    variance nudges the score up.
  - *Vocabulary richness*: type-token ratio (unique words / total words).
    Low richness (repetitive vocabulary) nudges the score up.
  - Each sub-signal is normalized to 0–1 (`1 - clamp(coefficientOfVariation, 0, 1)`
    for burstiness, `1 - clamp(typeTokenRatio, 0, 1)` for vocabulary richness)
    then combined 50/50: `aiLikelihoodScore = round(100 * (0.5 * burstinessSignal + 0.5 * vocabSignal))`.
    Documented in-code as a rough heuristic, not a validated AI-detector.

- **`flaggedSentences`**: `[{ sentence, score, matchedSource }]`, where
  `matchedSource` is the matching corpus entry's `id`, or the string
  `"internal-duplicate"` when there's no corpus.

## `autoRewriteLoop(text, maxAttempts = 3, config = {})` — `workers/plagiarism-rewrite-loop.mjs`

`config`: `{ threshold = 30, db, workspaceId, model, corpus, apiKey?, provider?, sentenceThreshold? }`.
`db` / `workspaceId` / `model` are threaded straight into the existing
`callAnthropicForArticle(db, workspaceId, systemPrompt, userPrompt, model)`
from `workers/llm.mjs`, reused unchanged. `corpus` / `apiKey` / `provider` /
`sentenceThreshold` pass straight through to `checkOriginality`.

Per iteration:
1. `checkOriginality(currentText, config)`.
2. Record `{ attempt, before: { plagiarismScore, aiLikelihoodScore } }`.
3. Stop if `max(plagiarismScore, aiLikelihoodScore) <= threshold` or
   `attempt === maxAttempts`.
4. Otherwise, send all of `flaggedSentences` to Claude in **one batched
   call** — system prompt instructs a line-for-line rewrite (same order, same
   count, no commentary) that preserves meaning while reducing similarity/
   detectability. Parse the response by splitting on newlines.
   - If `callAnthropicForArticle` returns `{kind: 'unavailable', ...}`, or the
     returned line count doesn't match the flagged-sentence count, abort the
     loop early and return the best text obtained so far (`passed: false`).
     Never throws — matches the repo-wide "deterministic fallback, never a
     hard error" convention (D-186/D-190).
   - Otherwise splice the rewritten sentences back into `currentText` in
     place of the originals and continue to the next iteration.
5. Record `after` scores are captured by the next iteration's `before` (or,
   for the final iteration, by one last `checkOriginality` call after the
   splice so the report always has an `after` for every rewritten attempt).

Returns `{ finalText, finalScore, attempts, passed }`.

After the loop finishes (pass, fail, or abort), append one record to
`workers/config/plagiarism-rewrite-report.json`:
`{ timestamp, attempts: [{attempt, before, after, rewroteCount}], finalScore, passed }`.
Read-modify-write the whole array, same pattern as `logProviderUsage`.

## Testing — `workers/verify/plagiarismprobe.mjs`

- TF-IDF/cosine math sanity: identical sentences → similarity ≈ 1; unrelated
  sentences → similarity ≈ 0.
- `checkOriginality` with a corpus: a near-duplicate sentence is flagged,
  `plagiarismScore > 0`, `matchedSource` is the corpus id.
- `checkOriginality` with no corpus: an internally repeated sentence is
  flagged with `matchedSource: 'internal-duplicate'`.
- `aiLikelihoodScore`: a uniform-length, narrow-vocabulary sample scores
  higher than a varied, human-like sample (relative comparison, not an
  absolute pass/fail).
- Paid tier selected (e.g. `{ apiKey: 'x', provider: 'copyleaks' }`) →
  `unavailable: true`, no throw.
- `logProviderUsage` is called once per `checkOriginality` call (assert via
  an injectable log path, same pattern as `providersprobe.mjs`'s
  `TEST_LOG_PATH`).
- `autoRewriteLoop` with a fake Claude `fetchImpl` (same injection seam as
  `llmprobe.mjs`): score improves each call → stops before `maxAttempts`;
  score never improves → stops at `maxAttempts` with `passed: false`; Claude
  returns a mismatched line count → aborts early without throwing; report
  file gets one appended record per run (not overwritten).

  old `'none'` stub description/name to the new `local-tfidf` free entry.
