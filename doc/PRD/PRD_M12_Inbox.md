# PRD — M12: Inbox (Omnichannel)
**Layer:** L1 Core Ops | **Priority:** P0 | **Phase:** 1 (Sessions 12–13)
**Depends On:** M09, M05 (consent/A2P), M41, M04 | **Blocks:** M33 handoff, M32

## 1. Purpose
Every conversation with every contact in one thread — email, SMS, WhatsApp, FB/IG DM, live chat — with team assignment, AI auto-reply, and realtime updates. Implements original PRD Section 8 fully.

## 2. Core Features
(Original PRD Section 8 scope, plus wiring)
- **Three-panel layout:** conversation list (filters: status, channel, assignee, unread) / thread / contact context panel (profile summary, recent deals, quick tag/assign).
- **Channels:** two-way SMS (Twilio webhooks + send API — gated by `sms.canSend()` + `consent.check()` from M05); email (Gmail OAuth via M41 or SMTP; threading by Message-ID/References); WhatsApp (Meta Cloud API, 24h session rules + template messages); FB + IG DM (Graph webhooks); live chat widget (embeddable JS, visitor→contact capture, Pusher realtime).
- **Thread unification:** all messages for a contact in one conversation stream with per-message channel icons; reply channel selector defaults to last-inbound channel.
- **Team features:** assign dropdown; statuses Open / In Progress / Resolved / Spam; internal notes (purple, @mentions → M04); canned responses ("/" search + insert, variables like {{first_name}}).
- **AI auto-reply mode:** per-conversation toggle; replies generated by linked M33 agent (or default workspace agent); AI messages labeled; auto-disables on human reply; confidence handoff (M33 rules).
- **Missed call → SMS:** Twilio voice webhook → auto-text template.
- **Search:** Postgres full-text over messages, filtered by workspace.
- **Realtime:** Pusher — new messages, assignment changes, typing indicator (live chat).
- **Timeline + metering:** every send/receive → `timeline.add()`; sends metered (`sms.sent`, `email.sent`).

## 3. Database Schema
Original PRD Section 8 tables Prisma-ized (`Conversation, Message, Channel, CannedResponse`) with additions: `Message.aiGenerated Boolean`, `Message.status` (queued|sent|delivered|failed with provider callbacks), `Conversation.unreadCount`, tsvector index on Message.content.

|---|---|---|

| PATCH | /api/conversations/:id | Assign / status / AI-mode toggle |
| POST | /api/conversations/:id/notes | Internal note |
| CRUD | /api/canned-responses | Templates |
| POST | /api/webhooks/twilio/sms \| /voice | Inbound SMS / missed call |

| POST | /api/chat-widget/message | Live chat inbound |
| GET | /api/chat-widget/script/:workspaceId | Widget embed JS |
| GET | /api/conversations/search?q= | Full-text search |

## 5. UI
- /inbox: three panels; list virtualized; composer with channel selector, attachments (M06), canned "/" menu, note/reply toggle; AI-mode switch; keyboard shortcuts (r reply, e resolve, a assign)
- /settings/channels: connect email/Twilio number/WhatsApp/Meta pages, channel status
- Widget: floating bubble, branded, pre-chat name/email form

## 6. Acceptance Criteria
- [ ] Inbound SMS/WhatsApp/FB/IG create-or-append conversations correctly (contact match by phone/PSID/email)
- [ ] Consent + A2P gates block sends with actionable UI messages
- [ ] Email threading groups replies correctly
- [ ] AI auto-reply drafts via M33, labels messages, disengages on human reply
- [ ] Realtime <1s for new messages; unread counts accurate
- [ ] Delivery status callbacks update message state
